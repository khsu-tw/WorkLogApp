## 📋 技術細節

### API 變更對照

| 功能 | Supabase | PocketBase |
|------|----------|------------|
| 客戶端 | `create_client(url, key)` | `PocketBase(url)` |
| 認證 | API Key (無狀態) | Token (可選) |
| 查詢 | `sb.table(TABLE).select().execute().data` | `pb.collection(TABLE).get_full_list()` |
| 插入 | `sb.table(TABLE).insert(data).execute()` | `pb.collection(TABLE).create(data)` |
| 更新 | `sb.table(TABLE).update(data).eq("id", id).execute()` | `pb.collection(TABLE).update(id, data)` |
| 刪除 | `sb.table(TABLE).delete().eq("id", id).execute()` | `pb.collection(TABLE).delete(id)` |
| 健康檢查 | `/rest/v1/{table}?select=id&limit=1` | `/api/health` |

### 環境變數變更

## 🔄 從 Supabase 切換到 PocketBase

### 新用戶設定
1. 下載並啟動 PocketBase
2. 參考 POCKETBASE_SETUP.md 創建 collection
3. 啟動 Work Log 應用
4. 自動彈出設定對話框
5. 輸入 PocketBase URL: http://127.0.0.1:8090
6. 開始使用

### 現有 Supabase 用戶遷移
1. 備份現有數據
2. 設定 PocketBase
3. 運行 `python migrate_data.py`
4. 驗證數據完整性
5. 更新 .env 配置
6. 重啟應用

## ⚠️ 注意事項

### PocketBase 設定要點
1. **worklogs 欄位必須為 "Editor" 類型** (支援大型 base64 圖片)
2. **API Rules 需設為公開** (或使用 Token 認證)
3. **需自行啟動 PocketBase 服務器**

### 圖片數據
- ✅ 所有圖片以 base64 格式嵌入 worklogs 欄位
- ✅ 格式: `[IMG_B64:iVBORw0KGgo...]`
- ✅ Word/Excel/PDF 導出時自動轉換為圖片

### 同步機制
- 保持原有的離線優先架構
- 每 30 秒自動同步
- 衝突自動檢測和解決

**Q: 同步失敗？**
A: 檢查 PocketBase 是否運行在 http://127.0.0.1:8090

**Q: 遷移數據重複？**
A: 使用 fix_local_duplicates.py 清空本地，重新同步
